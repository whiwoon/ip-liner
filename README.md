# IP Liner

IP Liner는 감사·관리·스캔 대상 IP 목록을 브라우저 안에서 정리하는 단일 HTML 도구입니다.

- 서버 없음
- 설치 없음
- 외부 통신 없음
- 첨부 모드와 복사 붙여넣기 모드 지원
- 첨부 모드에서 파일 선택 및 드래그앤드롭 업로드 지원
- 엑셀 `.xlsx`/`.xls` 업로드 지원
- 전체 시트의 전체 셀에서 IP 후보 자동 추출
- 추출된 IP의 원본 시트/셀 위치 추적
- 단일 IP, CIDR, 하이픈/물결 범위, 쉼표/공백 혼합 입력 처리
- 처리 로그 최신 위치 자동 스크롤
- 기본 다크 모드와 화이트 모드 전환 지원

## 현재 화면 구성

IP Liner는 파일첨부/복사 붙여넣기 → 정리 방식 선택 → 정리 시작 후 결과/로그 확인 → 역추적 순서로 필요한 영역만 열리는 UI입니다. 범위 확장 방식은 정리 방식 선택 단계에서 대역별로 선택합니다.

기본은 다크 모드입니다. 상단에서 화이트 모드로 바꿀 수 있습니다. 화이트 모드는 종이색이 아닌 단순 흰색 배경을 사용합니다.

1. **파일 첨부 / 복사 붙여넣기**
   - 첫 화면에는 입력 방식만 보입니다.
   - 기본값은 **파일 첨부**입니다.
   - 파일은 버튼으로 선택하거나 드래그앤드롭할 수 있습니다.
   - 직접 입력하려면 **복사 붙여넣기** 탭으로 전환합니다.
2. **정리 방식 선택**
   - 입력값에서 찾은 IP 대역을 먼저 보여주고 스캔 대상에 포함할 대역을 선택합니다.
   - 기본적으로 모든 후보 대역이 선택되고 처리 방식은 **범위 확장 안함**으로 지정됩니다.
   - 상단의 **전체 선택**, **전체 해제**, **범위 확장 안함**, **입력구간 확장**, **대역 전체 추가** 버튼으로 대역 선택과 처리 방식을 조정합니다.
   - 목록에 없는 스캔 대상 대역은 수동 스캔 대상 대역에 CIDR로 넣을 수 있습니다.
   - 수동 스캔 대상 대역 아래의 **정리 시작** 버튼으로 실행합니다.
3. **진행률 / 처리 로그**
   - 실행을 누른 뒤에만 나타납니다.
   - 진행률과 처리 로그는 별도 영역으로 분리되어 있습니다.
   - 로그는 새 내용이 추가될 때 최신 로그 위치로 자동 스크롤됩니다.
   - 새 영역이나 버튼이 나타날 때 해당 위치로 자동 스크롤됩니다.
4. **정리된 IP 목록**
   - 완료 후 나타납니다.
   - 목록 영역은 일정 높이를 넘으면 내부 스크롤로 봅니다.
   - **결과 복사**와 **TXT 다운로드** 버튼은 영역을 반씩 나누어 사용합니다.
   - TXT 다운로드 파일명에는 다운로드 시점의 날짜와 시간이 들어갑니다. 예: `ip-liner-targets-20260623-143012.txt`
   - 다운로드 후 버튼 하단에 다운로드된 파일명이 표시됩니다.
   - 결과 복사 후 버튼 하단에 클립보드 복사 완료 문구가 표시됩니다.
5. **역추적**
   - 완료 후 나타납니다.
   - IP를 입력하면 **출처 찾기** 버튼이 입력란 하단에 나타납니다.
   - 출처 찾기 버튼은 소속 영역의 가로폭 전체를 사용합니다.

## 사용 방법

1. `ip-liner.html`을 브라우저로 엽니다.
2. 기본값인 **파일 첨부** 모드에서 파일을 선택하거나 드래그앤드롭합니다. 직접 입력하려면 **복사 붙여넣기** 모드로 바꿉니다.
3. **정리 방식 선택**에서 스캔 대상 대역과 대역별 처리 방식을 확인합니다. 기본값은 전체 대역 선택 + **범위 확장 안함**입니다.
4. 필요하면 상단 버튼으로 전체 선택/해제하거나, 체크한 대역에 **범위 확장 안함**, **입력구간 확장**, **대역 전체 추가**를 일괄 적용합니다.
5. 목록에 없는 스캔 대상 대역이 있으면 수동 스캔 대상 대역에 CIDR로 입력합니다.
6. 수동 스캔 대상 대역 아래의 **정리 시작**을 누릅니다.
7. 분리된 **진행률** 영역과 **처리 로그** 영역을 확인합니다.
8. 자동으로 해석할 수 없는 값이 있으면 수동 입력 창에서 대체 IP 목록을 입력하거나 비워서 제외합니다.
9. 완료 후 나타나는 정리된 IP 목록을 복사하거나 TXT로 다운로드합니다.
10. 완료 후 나타나는 역추적 입력란에 스캔 결과 IP를 입력하고 **출처 찾기**를 누르면 원본 위치를 확인할 수 있습니다.

## 입력값 정리

입력값 정리는 사용자가 입력한 IP 관련 문자열을 최종 처리 가능한 개별 IP 목록으로 바꿉니다.

자동 처리:

- 단일 IP: `192.168.1.10`
- CIDR: `192.168.1.0/24`
- 호스트/프리픽스 표기: `12.123.12.201/20` → `12.123.0.0/20`처럼 네트워크 CIDR로 정규화
- 하이픈 범위: `192.168.1.10-192.168.1.20`, `192.168.1.10-20`
- 물결 범위: `192.168.1.10~192.168.1.20`, `192.168.1.10~20`
- 쉼표, 공백, 줄바꿈으로 구분된 여러 값
- `http://192.168.1.10`, `192.168.1.10:443`처럼 IP가 포함된 값

처리 규칙:

- CIDR과 범위 표현은 개별 IP 목록으로 펼칩니다.
- CIDR 문법은 맞지만 host bit가 켜져 있으면 호스트/프리픽스 표기로 보고 네트워크 주소로 정규화하며, 처리 로그에 남깁니다.
- CIDR은 일반적으로 사용 가능한 호스트 IP만 포함합니다. `/32`는 해당 IP 1개, `/31`은 두 IP를 모두 포함합니다.
- 중복 IP는 제거하고, IP 숫자 기준 오름차순으로 정렬합니다.
- 잘못된 IP, 잘못된 CIDR, 해석할 수 없는 문구, 서브넷 마스크, 예약 주소처럼 호스트 IP가 아닌 값은 정리 방식 선택 단계에서 자동 제외 항목으로 표시합니다.

## 정리 방식 선택

파일 첨부나 복사 붙여넣기 후, IP Liner는 입력값에서 스캔 대상 후보 대역을 먼저 찾고 정리 방식을 고를 수 있게 합니다.

이 단계에서 할 수 있는 일은 다음과 같습니다.

- 입력값에서 찾은 대역은 기본적으로 모두 체크됩니다.
- 기본 처리 방식은 **범위 확장 안함**입니다.
- **전체 선택** / **전체 해제** 버튼으로 후보 대역을 한 번에 조정합니다.
- 체크한 대역에는 **범위 확장 안함**, **입력구간 확장**, **대역 전체 추가** 버튼으로 같은 처리 방식을 한 번에 적용할 수 있습니다.
- 목록에 없는 스캔 대상 대역은 **수동 스캔 대상 대역**에 CIDR, 단일 IP, 범위로 입력합니다.
- 서브넷 마스크, 예약 주소, 잘못된 IP처럼 호스트 IP가 아닌 값은 이 단계에서 자동 제외 항목으로 표시됩니다.

예:

```text
12.123.0.0/16
100.64.0.0/10
```

선택된 대역 안에 들어가는 입력값만 정리 대상이 되며, 최종 출력 범위는 대역별 처리 방식에 따라 달라집니다.

## 대역별 처리 방식

입력값 정리 후 생성된 개별 IP 목록을 기준으로 최종 출력 범위를 추가 확장할지 대역별로 선택합니다. 출력은 항상 개별 IP 목록입니다.

- **범위 확장 안함**: 입력값 정리로 생성된 IP만 사용합니다. 예: `192.168.1.10`, `192.168.1.20` → 두 IP만 출력합니다.
- **입력구간 확장**: 같은 `/24` 대역 안에서 가장 작은 IP부터 가장 큰 IP까지 포함합니다. 여러 `/24` 대역이 있으면 대역별로 각각 적용합니다. 예: `192.168.1.10`, `192.168.1.20` → `192.168.1.10~20`을 출력합니다.
- **대역 전체 추가**: 정리된 IP가 하나라도 포함된 `/24` 대역 전체, 즉 `.0`부터 `.255`까지를 최종 출력에 포함합니다. 예: `192.168.1.10` → `192.168.1.0~255`를 출력합니다.

## 엑셀 추출과 역추적

엑셀 파일을 올리면 IP Liner는 양식을 해석하려고 하지 않고, 모든 시트의 모든 셀에서 IP 후보를 찾습니다. 후보마다 원본 위치를 함께 저장합니다. 암호화된 엑셀 파일은 직접 처리하지 않고 비밀번호를 해제해 저장한 뒤 다시 첨부하라는 안내를 표시합니다.

예:

```text
Audit Targets!B4 = 192.168.20.1-5
Firewall Scope!B3 = 10.20.1.10-10.20.1.20
```

정리 후 스캔 결과 IP를 입력하면 그 IP가 어느 원본 시트/셀 값을 통해 결과에 포함됐는지 보여줍니다. **입력구간 확장**이나 **대역 전체 추가** 때문에 원본에는 없지만 생성된 범위 안에 포함된 IP도 구분해서 표시합니다.

## 제거된 기능

다음 기능은 현재 UI에서 제거되었습니다.

- 출력 형식 선택
- 화면 미리보기 줄 수 선택

대신 결과 영역에는 최종 결과 전체를 표시하고, 입력 오류와 처리 결과 요약은 계속 보여줍니다.

## 테스트용 샘플

`samples/sample-5000.txt`에 5,000줄짜리 텍스트 샘플이 들어 있습니다.

`samples/sample-excel-targets.xlsx`에는 여러 여러 입력 양식을 흉내낸 엑셀 샘플이 들어 있습니다. 다중 시트, 문장 속 IP, 쉼표/공백 혼합, 축약/전체 범위, CIDR, 잘못된 IP, 중복 IP, 큰 단일 IP 목록을 포함합니다.

## 로컬 테스트

```bash
node tests/browser-smoke.mjs
```

테스트는 로컬 Chromium의 headless 모드로 `ip-liner.html`을 열고 주요 기능을 확인합니다.

현재 브라우저 스모크 테스트에서 확인하는 항목:

- 파일명이 `ip-liner.html`이고 제목과 화면 이름이 **IP Liner**인지
- 기본 화면이 파일 첨부 모드인지
- 정리 방식 선택에서 기본적으로 전체 대역이 선택되고 범위 확장 안함이 적용되는지
- 체크박스로 스캔 대상 대역을 제외하면 결과에서 빠지는지
- 수동 스캔 대상 대역을 함께 사용할 수 있는지
- 드래그앤드롭 후에는 정리 방식 선택만 열리고 진행률/로그는 아직 숨겨져 있는지
- 초기화 후 같은 파일을 다시 선택하거나 드래그앤드롭해도 입력이 다시 반영되는지
- 수동 스캔 대상 대역 아래에 정리 시작 버튼이 있는지
- 진행률과 처리 로그가 분리되어 있는지
- 처리 로그가 최신 로그 위치로 자동 스크롤되는지
- 버튼 너비가 의도대로 적용되는지
- TXT 다운로드 파일명에 날짜/시간이 들어가고, 다운로드/복사 상태 문구가 표시되는지
- CIDR 정규화, 범위 확장, 대역 전체 추가, 수동 입력, 엑셀 추출, 역추적이 동작하는지
